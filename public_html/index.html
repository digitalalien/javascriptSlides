What is Javascript?

Javascript is an easy-to-use programming language that can be embedded in the header of your web pages. It can enhance the dynamics and interactive features of your page by allowing you to perform calculations, check forms, write interactive games, add special effects, customize graphics selections, create security passwords and more.

Java vs. Javascript

Actually, the 2 languages have almost nothing in common except for the name. Java is not considered an easy-to-use language for non-programmers.
Javascript is much simpler to use than Java. With Javascript, if I want check a form for errors, I just type an if-then statement at the top of my page. No compiling, no applets, just a simple sequence.

Variables

Variables are used to store values.

Variables can be defined using multiple var statements or in a single combined statement.

Declare then initialize 

    var y; //value is undefined
    y = 5; //value is 5

Declare and initialize in one statement 

    var y = 5; //value is 5
    console.log(y);

Variables can also store the results of expressions. 

    var y = 5+5; //y = 10
    var x = 5*y; //x = 50

    var name = "Andrew";
    var welcomeMessage = 'Hello ' + name; //welcomeMessage = 'Hello Andrew'
Variable Names
  • Variable names must begin with a letter, $ or _
  • Variable names are case sensitive (g and G are not the same)
  • Avoid reserved words (words used by Javascript ie: var, for, else)
  • Choose meaningful words
  • camelCase is preferred to under_score
  • Be consistent with your naming conventions
Primitive Types

  • String: Strings are wrapped in single or double quotation marks.

        var welcome = "Welcome to learning JavaScript";
    var location = "Velocity Southern Indiana";

  • Number: Numbers are positive or negative numeric values and include floating point values.

        var pizzas = 8;
    var cost = 8.99;

  • Boolean: Booleans are either true or false

        var iLikeCats = false;
    var iLikeDogs = true;

  • null: Null represents the absence of a value

        var goodPlaceToEat = null;

  • undefined: Undefined represents a value that has not been defined

        var gameSystem;

Arrays

Arrays are zero-indexed, ordered lists of values. They are used to store a set of related items of the same type, thought they can include multiple types of items, including other arrays.

You can create an array using either the object constructor or the literal declaration. You can then assign the variables a list of values after the declaration. 


	// Simple array with constructor
	var constructorArray = new Array("Hello", "World");
	// Literal Declaration, this is the preferred way to create an array
	var literalArray = ["Hello", "World"];

Using the literal declaration is preferred as the object constructor can be error prone due to its arguments.

Conditional Code

Using if and else statements we can run code if certain conditions are met. 

<script>
	var foo = true;
	var bar = false;
	
	if( bar ){ //Check if bar is true
		//This code will never run because bar === false
		console.log( "I wont run!" );
	}
	
	if( foo ){ //Check if foo is true
		//This code will run because foo is true
		console.log( "I am true!" );
	}
	
</script>
Loops

We create loops in order to run certain code a number of times.

The for loop loops over the code block until the condition (i < 5) becomes false. 

<script>
	for( var i = 0; i < 5; i++ ){
		console.log("Loop #: " + i);
	}	
</script>

The while loop will execute the code body until it's condition evaluates to false. 

<script>
	var i = 0;
	while( i < 5 ){
		console.log("While #: " + i);
		i++;
	}	
</script>
Wait! These seem the same! Which one should I use!?
It's really up to you, neither is technically right or wrong.
You will learn to use whichever one best fits the situation or your preference.
Popup Boxes

Alert: alert("I am an Alert Box!");

Alert boxes are often used to provide information to the users. As a developer you can use them to help debug code.

Prompt: prompt("I am a Prompt Box! Tell me your name!");

A prompt box can be used if you want the user to input a value before entering a page.

Confirm: confirm("I am a Confirm Box! Continue?");

A confirm box can be used if you want the user to verify or accept something.

Functions

Functions contain re-usable blocks of code. Functions can take zero or more arguments and can return a value.

Creating a function

function myNameIs(){
    console.log('Andrew');
}
OR 
var myNameIs = function(){
    console.log('Andrew');
};

Calling a function

myNameIs();
Function Parameters

You can pass any number of named arguments to a function.

function myNameIs(name){
    console.log('Hi! My name is ' + name);
}

myNameIs('Andrew');
myNameIs('Derrick');

function calculateTotal(price, qty){
    var total = price * qty;
    console.log(total);
}

calculateTotal(3.89, 8);
calculateTotal(4.34, 3);
Returning values from a function

The return keyword will return a value from the function and exit the function.

function calculateTotal(price, qty){
    var total = price * qty;
    return total;
}

var order1 = calculateTotal(3.89, 8);
var order2 = calculateTotal(4.34, 3);

var ordersTotal = order1 + order2;

//you could even do this
var ordersTotal = calculateTotal(3.89, 8) + calculateTotal(4.34, 3);

Events

Events are how we trigger our functions to run. The easiest example is a button, whose definition includes the words onClick="run_my_function()".

Events allow us to do things like:

  • Form Validation
  • Games
  • Interactive Photo Galleries
  • Lots more things that involve user interaction.

Event types include but are not limited to:

  • Mouse Events - click, dblclick, mouseup, mousedown, etc.
  • Keyboard Events - keyup, keydown, keypress
  • Form Events - onblur, onchange, onfocus, onsubmit, etc
  • Touch Events - touchstart, touchend, touchmove, touchenter, etc.
Adding Event Listeners

You can set certain functions to run when an event is triggered by the user. These functions will "listen" for the event and then initiate the function.

addEventListener works in these major browsers

Note: The addEventListener method is supported in Internet Explorer since Version 9

<button id="clickMe">Please Click ME!</button>
<script>
    function clickedMe(){
        console.log('You Clicked Me!');
    }
    var clickMeButton = document.getElementById('clickMe');
    clickMeButton.addEventListener('click', clickedMe, false);
</script>
